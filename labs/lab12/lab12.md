# Теоретическое введение

**Настройка сети в Linux** является важнейшей задачей системного администрирования. Современные дистрибутивы используют различные инструменты для управления сетевыми параметрами.

## Основные сетевые конфигурационные файлы:

- **/etc/network/interfaces** — классический файл настроек сети (Debian/Ubuntu)
- **/etc/sysconfig/network-scripts/** — директория с файлами интерфейсов (RHEL/CentOS/Rocky)
- **/etc/resolv.conf** — настройки DNS
- **/etc/hosts** — локальное разрешение имён
- **/etc/hostname** — имя хоста
- **/etc/nsswitch.conf** — порядок разрешения имён

## Основные команды для работы с сетью:

- `ip` — современная утилита для настройки сетевых интерфейсов
- `ifconfig` — классическая утилита (устаревшая)
- `nmcli` — управление NetworkManager из командной строки
- `nmtui` — текстовый интерфейс для NetworkManager
- `ping` — проверка доступности узлов
- `ss` / `netstat` — просмотр сетевых соединений
- `route` / `ip route` — управление маршрутизацией
- `hostname` / `hostnamectl` — управление именем хоста
- `systemctl` — управление сетевыми службами

# Основные сетевые команды

| Команда | Назначение | Пример |
|---------|------------|--------|
| `ip addr show` | Просмотр IP-адресов | `ip addr show eth0` |
| `ip route show` | Просмотр маршрутов | `ip route show` |
| `ss -tulpn` | Просмотр открытых портов | `ss -tulpn | grep 80` |
| `nmcli dev status` | Статус устройств | `nmcli dev status` |
| `hostnamectl` | Управление именем хоста | `hostnamectl set-hostname server` |
| `systemctl restart network` | Перезапуск сети | `systemctl restart network` |
| `ping` | Проверка связи | `ping -c 4 google.com` |
