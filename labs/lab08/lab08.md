# Теоретическое введение

**Планирование задач** является неотъемлемой частью администрирования Linux. Системы cron и at позволяют автоматизировать выполнение задач в определенное время.

## Cron — регулярное выполнение задач

Cron используется для выполнения задач по расписанию (ежедневно, еженедельно, ежемесячно и т.д.). Основные компоненты:

- **crontab** — файл с расписанием задач для каждого пользователя
- **/etc/crontab** — системный файл расписания
- **/etc/cron.d/** — директория с дополнительными файлами расписания
- **/etc/cron.hourly/**, **/etc/cron.daily/**, **/etc/cron.weekly/**, **/etc/cron.monthly/** — директории для скриптов

## At — однократное выполнение задач

At используется для выполнения задач один раз в указанное время. Основные команды:

- **at** — запланировать задачу
- **atq** — просмотр очереди задач
- **atrm** — удаление задачи из очереди
- **batch** — выполнение задачи при низкой нагрузке системы

# Сравнение cron и at

| Характеристика | cron | at |
|----------------|------|-----|
| Тип выполнения | Регулярное (периодическое) | Однократное |
| Синтаксис времени | Поля (минуты, часы, дни, месяцы) | Естественный язык (now + 2 hours) |
| Просмотр задач | `crontab -l` | `atq` |
| Удаление задач | `crontab -e` (удалить строку) | `atrm <номер>` |
| Системные директории | /etc/cron.d/, /etc/cron.daily/ и др. | Нет |
| Подходит для | Регулярных задач (бэкапы, обновления) | Разовых задач (напоминания, разовые операции) |
